# ADR-003: エラーを出したらログも出す

## ステータス
メンバー間で合意済み

## コンテキスト
エラーハンドリングをしている全ての箇所でログを出力する。各層で再ログを出すことは許容する。

ロガーは ADR-002 に従い `slog`。ctx があれば `slog.ErrorContext(ctx, ...)`。

```go
if err != nil {
    slog.ErrorContext(ctx, "failed to find project", "error", err) // PII は ADR-004 参照
    return err
}
```

NotFound のログレベル:

| 状況 | レベル | 例 |
|---|---|---|
| 想定外（不整合・バグの兆候） | `slog.Warn` | タスク作成時に指定した `project_id` が見つからない |
| ユーザー操作起因（想定内） | `slog.Debug` | URL に存在しない `project_id` を指定 |
| 正常系（重複チェック等） | 出さない | INSERT 前の存在確認 |

### 例外
- 呼び出し先が既にログを出している場合、同層内では二重出力を避ける（層をまたぐ場合は再出力 OK）
- 単体テストには適用しない

## 理由
本番運用時にログ不足で原因特定が困難になることを避ける。各層の再ログにより、伝播過程で context（リクエスト URL・操作種別など）を付与しつつ追跡できる。
